## CS 305 Module Eight Journal Reflection

### Client Summary
Artemis Financial is a financial services company that provides personalized financial planning solutions. The company needed to improve the security of its web application to protect sensitive customer and financial data. The main goal was to identify vulnerabilities and implement secure communication mechanisms.

### What I Did Well & Importance of Security
I successfully identified vulnerabilities in the application and applied secure coding practices to fix them. Secure coding is important because it protects sensitive data, prevents cyberattacks, and ensures system reliability. Software security adds value by increasing customer trust and preventing financial and reputational loss.

### Challenges & Helpful Parts
One challenging part was understanding the dependency-check results and determining which vulnerabilities were critical. This process was helpful because it improved my ability to analyze and prioritize security risks.

### Increasing Security Layers
I improved security by implementing HTTPS using SSL certificates and adding SHA-256 hashing for data integrity. I also used dependency-check to identify vulnerabilities. In the future, I would use tools like static analysis and penetration testing to further assess security.

### Ensuring Functional & Secure Code
I tested the application after making changes to ensure it worked correctly. I verified secure communication by running the application over HTTPS and testing the hash endpoint. I also reran dependency-check to ensure no new vulnerabilities were introduced.

### Tools & Practices Used
- OWASP Dependency-Check  
- Java Keytool  
- SHA-256 hashing  
- HTTPS/SSL configuration  
- Secure coding practices  

### What I Can Show Employers
I can demonstrate my ability to identify vulnerabilities, implement secure communication, and apply industry-standard security practices. This project shows my skills in secure software development and testing.
